# NightTable CO v1.6.0

**Production-ready packaging release** of the Colombia Food & Nightlife marketplace + venue SaaS.

**Tag:** `v1.6.0`  
**Repo:** https://github.com/leonardeco/saas-boilerplate

## Highlights

- Full product stack: auth (cookies + OAuth), premium catalog, booking, claims, reviews, Stripe, ingestion worker
- Production hardening: weak secrets rejected, HSTS, rate limits, Swagger off in prod
- Deploy: Render blueprint, Railway Docker, multi-stage Dockerfiles
- Mobile: PWA + Capacitor shell
- Observability: `/metrics`, OTLP lite, region labels
- Legal stubs: `/privacy`, `/terms`

## Quick deploy

### Option A — Render

1. New → Blueprint → this repo (`render.yaml`)
2. Fill secrets (`WEB_URL`, `CORS_ORIGIN`, `API_PUBLIC_URL`, Stripe, etc.)
3. Migrate DB from your machine with Render `DATABASE_URL`
4. Create SUPERADMIN

Guide: [docs/runbooks/deploy-render.md](../runbooks/deploy-render.md)

### Option B — Railway

1. Add Postgres + Redis plugins  
2. Deploy api / worker / web from Dockerfiles at monorepo root  
3. Set env from `.env.production.example`  
4. Migrate + seed + superadmin  

Guide: [docs/runbooks/deploy-railway.md](../runbooks/deploy-railway.md)

## Post-deploy

```bash
npm run db:migrate -w @saas/db
npm run db:seed -w @saas/db
npm run make-superadmin -- ops@yourdomain.com
curl -sf https://api…/health
curl -sf https://api…/ready
```

Full checklist: [docs/runbooks/release-checklist.md](../runbooks/release-checklist.md)

## Breaking / important notes

- `NODE_ENV=production` **requires** strong JWT secrets, `COOKIE_SECURE=true`, https `WEB_URL`, non-localhost `CORS_ORIGIN`
- Demo venues from seed are for staging; use ingestion for production catalog
- Worker should run **scale 1** unless you redesign queue consumers
- Replace placeholder emails in privacy/terms before public launch
